## What This Project Is

This is a `dotnet new` template that generates an opinionated starter solution for Umbraco v16 packages. It is **not** a typical application — it's a template packaging project. The generated output includes a package project, a test site, GitHub Actions, issue templates, and NuGet configuration.

Package ID: `Umbraco.Community.Templates.PackageStarter`
Template short name: `umbracopackagestarter`

## Build & Pack

```bash
# Pack the template into a NuGet package (version required)
dotnet pack .\template-pack.csproj -c Release /p:Version=0.1.0
```

There are no tests, no lint commands, and no build step beyond packing. The project file (`template-pack.csproj`) is of type `Template` — it bundles the `template/` directory contents into a `.nupkg`.

## Local Development Workflow

```bash
# Install the template from the local template/ directory
dotnet new install .\template

# Uninstall (must use full path for locally-installed templates)
dotnet new uninstall <full-path-to-repo>\template

# Generate a test package to verify changes
dotnet new umbracopackagestarter -n TestPackage -an "Test Author" -gu "testuser" -gr "test-repo"
```

After modifying template files, you do **not** need to reinstall unless you changed the symbols/parameters in `template/.template.config/template.json`.

## Architecture

### Root level
- `template-pack.csproj` — The packaging project. Bundles everything under `template/` into a NuGet template package. Not a buildable application.
- `docs/` — NuGet readme and logo assets included in the package.

### template/ directory (the actual template content)
This is the directory that gets installed as a `dotnet new` template. Everything here is subject to token replacement when a user generates a new project.

- `.template.config/template.json` — Template engine configuration. Defines parameters (symbols), token replacements, post-actions, and source transforms. Key symbols:
  - `authorName` (-an) → replaces `AUTHOR_NAME`
  - `githubUser` (-gu) → replaces `GITHUB_USERNAME`
  - `githubRepo` (-gr) → replaces `GITHUB_REPOSITORY`
  - `packageTitle` (-pt) → replaces `PACKAGE_TITLE` (auto-derived from name via PascalCase splitting if omitted)
  - Auto-generated: `currentYear`, `httpPort`, `httpsPort`
  - `sourceName: "PackageStarter"` — this string gets replaced with the user's `-n` value throughout all file names and contents.

- `setup.cmd` — Post-action script that runs after template generation. Initializes git, installs latest Umbraco templates, creates the package project using the core `umbraco-extension` template, replaces .csproj with nuget-configured version, wires up the solution.

- `src/PackageStarter.TestSite/` — Pre-configured Umbraco test site (SQLite, unattended install, uSync). References the package project locally.

- `src/PackageStarter/PackageStarter_nuget.csproj` — The nuget-ready .csproj that replaces the one generated by `umbraco-extension`. Contains NuGet metadata, package configuration, and Umbraco dependencies.

### Release process
Pushing a semver tag (e.g. `1.0.0`) triggers `.github/workflows/release.yml`, which packs and pushes to NuGet using the `NUGET_API_KEY` secret.
